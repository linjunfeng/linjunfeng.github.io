WooCommerce 产品图片优化思路

WooCommerce 是 wordpress 的电子商务插件。 公司最近打算用这个来wordpress 来建站。 在导入数据的时候发现非常的慢， 几千条好好要几个小时。

通过分析，发现wordpress 在处理图片的时候会生成很多不同尺寸的图片（基本上没用到），而一个产品好几张图片， 就要生成几十的图片，猜想是不是以为生成图片而导致速度慢。于是我把生成缩略图去掉

<?php

add_filter('intermediate_image_sizes_advanced', function () {
    return false;
}, 100);

测试下来，缩略图去掉导入速度有所提升，但还是有点慢。后来发现原来wordpress 在处理图片的时候， 图片也是一条 post 记录 类型为attachment， 这样表记录增加了好几倍。但很多情况下产品图片仅仅是原来展示的， 明显没必要。 于是我就打算弃用 wordpress 默认图片字段。 改用自定义字段定义图片规则， 再获取图片的时候通过钩子来hack。测试下来速度明显提升好几倍。
